# Cross-Domain Query Samples

멀티 에이전트 시스템의 크로스 도메인 쿼리 예시 5건입니다.
WMS(창고), TMS(운송), FMS(차량), TAP(호출) 도메인을 조합합니다.

## 1. FMS + TMS (차량 정비 → 배차 제외)

> "정비 중인 차량을 오늘 배차에서 제외해줘"

- **FMS**: 정비 중(`maintenance`) 상태 차량 목록 조회
- **TMS**: 해당 차량을 배차 스케줄에서 제외

## 2. WMS + TMS (재고 → 출고 운송)

> "재고가 부족한 SKU를 보유한 창고로 긴급 배송 요청해줘"

- **WMS**: 재고 부족(`quantity < threshold`) SKU 및 보유 창고 조회
- **TMS**: 해당 창고로의 긴급 배송 건 조회/생성

## 3. TMS + TAP (배송 → 라스트마일 호출)

> "오늘 배송 완료된 건 중에서 고객 호출 요청이 있는 건 알려줘"

- **TMS**: 오늘 `delivered` 상태 배송 건 조회
- **TAP**: 해당 고객의 호출 요청(`CallRequest`) 매칭

## 4. FMS + WMS (차량 → 창고 적재)

> "물류센터별로 현재 대기 중인 차량 수와 창고 적재율을 비교해줘"

- **FMS**: 물류센터별 `active` 상태 차량 수 조회
- **WMS**: 각 창고의 빈(`Bin`) 적재율(`occupied/total`) 조회

## 5. TMS + FMS + TAP (운송 + 차량 + 호출 3개 도메인)

> "연비가 낮은 차량이 담당하는 배송 건의 고객 피드백 평점을 확인해줘"

- **FMS**: 연비(`fuelEfficiency`) 하위 차량 목록 조회
- **TMS**: 해당 차량이 배정된 배송 건 및 고객 조회
- **TAP**: 해당 고객의 피드백(`Feedback`) 평균 평점 조회
